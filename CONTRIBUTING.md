# Contribuindo com M-triosk-

Obrigado pelo interesse em contribuir.

## Como contribuir
1. Abra uma issue descrevendo proposta/bug.
2. Crie branch (`feature/...`, `fix/...`).
3. Faça commits pequenos e objetivos.
4. Abra PR com contexto técnico e plano de validação.

## Convenções iniciais
- Priorizar modularidade por domínio (`src/`).
- Documentar decisões em `docs/`.
- Incluir scripts reproduzíveis em `scripts/`.

## Checklist de PR
- [ ] Código formatado e legível.
- [ ] README/doc atualizados.
- [ ] Passos de execução/teste descritos.
